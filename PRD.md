# PRD — MVP дашборда «Две руки российской макро»
*Спека для сборки в Claude Code. Среда: WSL2 (Ubuntu) + Streamlit + SQLite.*

---

## 1. Цель
Один публичный дашборд, показывающий перетягивание каната между **фискальным вливанием** (бюджет вливает деньги) и **монетарным торможением** (ЦБ держит высокую ставку), плюс датчики результата (официальная инфляция, наблюдаемая инфляция, денежные агрегаты). Назначение — пруф под контент: ссылка «я собрал дашборд, зайдите посмотрите».

## 2. Главный принцип (не нарушать)
- Это **прокси и сопоставление, а не «настоящая инфляция»**. Дашборд не выдаёт секретное число инфляции — он показывает силы и разрывы.
- Любую расчётную «вменённую инфляцию» (например, из M2) рисуем **только с явной подписью о допущении** (V = const → сейчас завышает; «для справки»).
- Где данные засекречены/неполные — честно отмечаем (структура расходов бюджета закрыта с 2022). Непрозрачность — часть нарратива, а не баг.
- **VAT-инфляция в v1 не используется** (ставка НДС изменилась 20→22% с 01.01.2026 + снижен порог УСН — структурный разрыв). Если добавлять позже — только с обязательной аннотацией разрыва.

## 3. Что входит в v1 / что НЕ входит
**Входит:** загрузка ~10 рядов в SQLite, 3 блока (Газ / Тормоз / Результат) + шапка-overview, базовые трансформации, секция «Методология», деплой на Streamlit Cloud.
**НЕ входит (это «север», не сейчас):** Postgres, FastAPI, Next.js, модельная регрессия инфляции, алерты, BTC/on-chain, региональные срезы, авторазбор PDF/сложный скрейпинг. Не усложнять.

## 4. Стек и среда (WSL2)
- WSL2 Ubuntu; проект в `~/projects/rumacro-dashboard` (**не** в `/mnt/c/...` — тормоза и кривой hot-reload).
- Python 3.11+, окружение `python3 -m venv .venv` (или `uv`).
- Зависимости: `streamlit pandas requests plotly openpyxl python-dateutil`. SQLite — из stdlib.
- Графики: Plotly (`st.plotly_chart`) — нужны тултипы с источником и две оси.
- Деплой: **Streamlit Community Cloud** (бесплатно, публичный URL, автодеплой из GitHub).
- Кодировка: российские CSV часто **Windows-1251** → `pd.read_csv(..., encoding="cp1251")`. xlsx — через openpyxl.
- git + GitHub с первого коммита. В корне — `CLAUDE.md` (принципы из §2 + стек, чтобы у агента всегда был контекст).

## 6. Модель данных (SQLite, long-format)
```sql
CREATE TABLE series (
  id TEXT PRIMARY KEY,         -- 'key_rate', 'm2', 'budget_deficit', ...
  title TEXT, unit TEXT, freq TEXT,        -- 'D' | 'M' | 'Q'
  panel TEXT,                  -- 'gas' | 'brake' | 'result' | 'context'
  source_name TEXT, source_url TEXT,
  transform TEXT, notes TEXT
);
CREATE TABLE observations (
  series_id TEXT, date TEXT, value REAL,
  fetched_at TEXT, flag TEXT,              -- 'final'|'prelim'|'seed'|'revised'
  PRIMARY KEY (series_id, date)
);
```
Правила: храни **исходную частоту** как есть, не апсемплить молча; сырые ряды отделять от производных (производные считать на лету в `transforms.py`).

## 8. Производные ряды (`transforms.py`)
- YoY, MoM, скользящие 3м / 12м.
- `deficit_vs_plan` = накопленный дефицит ÷ годовой план (план-2026 ≈ **3,786 трлн ₽**).
- `velocity_proxy` = номинальный ВВП (annualized) ÷ M2 (частота Q — отметить).
- `naive_qtm_infl` = M2 YoY − реальный ВВП YoY — **рисовать только с подписью «допущение: V = const, сейчас завышает, для справки»**.
- `gap` = observed_infl − cpi_yoy (разрыв «ощущаемая vs официальная»).

## 9. Раскладка дашборда (Streamlit)
**Шапка (overview):** крупно — официальная инфляция vs наблюдаемая (+ их разрыв), ключевая ставка, M2 г/г с **полосой цели ЦБ 5–10%**, дата последнего обновления каждого блока.

**Три секции/вкладки:**
1. **Газ** — дефицит факт vs план, расходы, ликвидная часть ФНБ, вклад бюджета в M2.
2. **Тормоз** — ставка, ставки по вкладам, доля «замороженных» срочных депозитов, кредитование.
3. **Результат** — ИПЦ, наблюдаемая инфляция + ожидания, M2/M0, прокси скорости, (опц.) `naive_qtm` с дисклеймером.

**Глобально:** переключатель диапазона дат; тултип показывает значение, единицы, источник, дату обновления; внизу — секция **«Методология»** (что прокси, что засекречено, формулы, оговорки про скорость и про НДС).

## 10. Подход к обновлению
- `ingest/*` тянут «лёгкие» источники (ставка, курс, деньги) и пишут в SQLite; запуск вручную или по расписанию.
- `seed_loader.py` грузит ручные CSV для PDF-источников; пользователь дополняет их раз в месяц.
- У каждого ряда хранить `fetched_at` и `flag` (final / prelim / seed); в шапке показывать свежесть.

## 11. Порядок сборки для Claude Code (строго по шагам)
1. **Скаффолд:** репо, `.venv`, `requirements.txt`, `db.py` со схемой, `.gitignore`, `CLAUDE.md`, пустой `app.py` («Hello»). Убедиться, что `streamlit run app.py` стартует.
2. **Один лёгкий источник end-to-end:** `cbr_keyrate.py` → в SQLite → линия ставки в Streamlit. (быстрая видимая победа)
3. **Деньги:** `cbr_money.py` (M0/M2/M2X + факторы) + `cbr_fx.py`.
4. **Ручные ряды:** `seed_loader.py` + сиды `cpi` / `observed_inflation` / `budget` / `nwf`.
5. **Сборка вида:** `transforms.py` + три блока + шапка-overview + секция «Методология» с дисклеймерами.
6. **Деплой** на Streamlit Community Cloud; получить публичную ссылку.

## 12. Критерии готовности (MVP)
- Приложение стартует локально и задеплоено, есть публичный URL.
- На одном экране видно: официальная инфляция vs наблюдаемая (+ разрыв), ключевая ставка, рост M2 vs цель, дефицит факт vs план.
- Минимум по 2 ряда в каждом из трёх блоков, с источником и датой обновления в тултипе.
- Есть секция «Методология» с оговорками (прокси не оракул; V = const; НДС-разрыв; засекреченные расходы).
- Данные в SQLite; «лёгкие» источники тянутся скриптом, «тяжёлые» — из seed-CSV.

## 13. Готчи (не споткнуться)
- Кодировка **cp1251** у российских CSV; xlsx — через openpyxl.
- URL'ы госсайтов «плывут» — подтверждать перед хардкодом, дату проверки оставлять в комментарии.
- Частоты разные (D/M/Q) — не апсемплить молча; скорость денег — квартальная.
- Данные пересматриваются задним числом — хранить дату загрузки и флаг `revised`.
- Файлы проекта держать в файловой системе WSL (`~/...`), не на `/mnt/c`.
- Не тащить сюда большой исходный PRD — он бэклог/«север», не план сборки v1.
