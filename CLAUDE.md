# CLAUDE.md — rumacro-dashboard

Контекст проекта для Claude Code. Полная спека и порядок сборки — в `PRD.md`. Этот файл — короткие правила, действующие всегда.

## Что это
Публичный дашборд «Две руки российской макро»: показывает противостояние **фискального вливания** (бюджет) и **монетарного торможения** (ЦБ) + датчики результата (инфляция официальная/наблюдаемая, денежные агрегаты). Это пруф под контент, не торговый терминал.

## Стек (не отклоняться)
- Python 3.11+, **Streamlit**, **SQLite** (stdlib), pandas, plotly, requests, openpyxl.
- Деплой — Streamlit Community Cloud. Графики — Plotly (`st.plotly_chart`).

## Среда
- WSL2 Ubuntu. Проект в `~/projects/...`, **не** в `/mnt/c`.
- Российские CSV часто в кодировке **cp1251** → читать с `encoding="cp1251"`. xlsx — openpyxl.
- git с первого коммита; `data/*.db` — в `.gitignore`.

## Принципы (НЕ нарушать)
1. **Прокси, не оракул.** Дашборд не вычисляет «настоящую инфляцию» — он показывает силы и разрывы между официальными рядами.
2. «Вменённую инфляцию» (например, из M2) — **только с явной подписью о допущении** (V = const → сейчас завышает; «для справки»).
3. НДС как измеритель инфляции **не используем** (ставка 20→22% с 01.01.2026 — структурный разрыв). Добавлять позже — только с аннотацией разрыва.
4. Засекреченные/неполные данные (структура расходов бюджета — с 2022) **честно отмечать**. Непрозрачность показываем, не маскируем.
5. Каждый график несёт источник, единицы и дату обновления (в тултипе).

## Конвенции
- SQLite long-format: `series` (метаданные) + `observations` (`series_id, date, value, fetched_at, flag`).
- Хранить **исходную частоту** (D/M/Q), не апсемплить молча. Сырые ряды отделять от производных (производные — в `transforms.py`).
- У каждого наблюдения — `fetched_at` и `flag` (`final|prelim|seed|revised`).
- **URL'ы госсайтов плывут**: перед хардкодом подтвердить актуальный путь/формат, дату проверки оставить в комментарии.
- Тяжёлые источники (PDF: наблюдаемая инфляция, ожидания, бюджет, ФНБ) на v1 — из `data/seed/*.csv`, обновляются вручную.
- Состояние UI — через `st.session_state`.

## Чего НЕ делать
- Не тащить тяжёлый стек: **никакого Postgres / FastAPI / Next.js**.
- Не строить модельную регрессию инфляции, алерты, BTC/on-chain, региональные срезы — это «север», не сейчас.
- Не скрейпить сложные PDF/порталы ради v1 — вместо этого seed-CSV.
- Не раздувать сверх MVP: если фичи нет в PRD (v1) — сначала спросить.

## Порядок работы
Идти по разделу «Порядок сборки» из PRD: скаффолд → один лёгкий источник end-to-end (ключевая ставка ЦБ) → деньги → ручные ряды → три блока + методология → деплой. После каждого шага — рабочий, запускаемый `streamlit run app.py`.
